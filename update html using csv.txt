How the HTML parses the CSV
Step 1 — At build time (Python):
The Python script reads your companies.csv file, converts every byte into a base64 string (~98,000 characters), and physically pastes that string into the HTML file as a JavaScript variable:
jsvar CSV_B64 = "aWQsbmFtZSxwb3N0YWwscHJvdmluY2U...";  // ← your entire CSV baked in
Step 2 — At runtime (Browser):
When you open the HTML, JavaScript runs atob(CSV_B64) which decodes the base64 back into the original CSV text, then a custom parser splits it by newlines and commas (handling quoted fields) to build 100 JavaScript objects — one per company.

⚠️ No — updating the CSV won't update the HTML
The CSV is baked into the HTML at the moment the file was generated. The HTML file and the CSV file are now completely disconnected — you could delete the CSV and the HTML would still work fine.
To update the HTML with new CSV data, you need to re-run the build script.
